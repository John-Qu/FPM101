# 需求澄清演练场景卡

## 场景A：商超鲜肉MAP
- 目标：货架期+低退货+陈列美观
- 需记录：
  - 品类与切割形态、渗液管理
  - 残氧目标与抽检频率、泄漏测试方法
  - 托盘材质与尺寸范围、膜防穿刺与防雾需求
  - 产能（托盘/分）、道数组合、换型频率与时间窗
  - 上下游接口（灌装/贴标/检漏）、追溯与标签规范
- 风险提示：膜穿刺与密封不良、纸基耐潮与强度、换型影响OEE
- 验收映射：节拍与停机原因、残氧≤目标、泄漏率达标、清洗时长

## 场景B：中央厨房预制菜VSP
- 目标：呈现与保形、批次一致、快速换型
- 需记录：
  - SKU与规格变更频率、刀具与格式切换需求
  - 纸基兼容与切边质量、膜贴合强度与耐穿刺
  - 产能与节拍、恢复稳定节拍时间
  - 清洗与卫生设计、占地与接口、数据采集（OEE/停机原因）
- 风险提示：纸基耐潮与分离便利、膜贴合导致穿刺、换型时长
- 验收映射：节拍+泄漏双测、切边与呈现一致性、清洗时长与卫生死角

## 场景C：即食餐常压/轻度MAP
- 目标：成本控制、运输稳健、标签与合规
- 需记录：
  - 托盘与盖膜匹配、加热方式（微波/烤箱）
  - 二次包装与码垛联动、冷链动线
  - 标签与追溯合规、能耗与备件寿命
- 风险提示：密封完整性与运输挤压、标签规范、冷凝与水汽管理
- 验收映射：密封一致性、装箱稳定性、标签与追溯完整

## 使用方法
- 选择一个场景进行角色扮演，按“需记录”逐项提问与记录
- 将风险提示转为测试计划，填入`templates/validation_checklist.md`
- 将记录归档到`templates/customer_requirements.md`并作为方案输入
